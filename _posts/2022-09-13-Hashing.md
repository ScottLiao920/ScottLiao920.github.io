title: Hashing Schemes
---

## Note: this hashing is not hashing in cryptography, it's only used inside DBMS to map a key to an integer. Hence only speed is concerned.

## **Static Hashing Schemes**

* The size of hash table is fixed. When maximum number of elements reached, a larger table (normally 2x original size) is required to rebuild from scratch. Empirically, to reduce the chance of key collision, the size of hash table should be twice as large as the key size.
* Linear Probe Hashing: Linearly searching for the nest free slot when collision happens.
* Robin Hood Hashing: Replace an occupied slot if it's too far away from original slot. Reinsert the replaced key.
* Cuckoo Hashing: Multiple hash tables are used. If no table have free slot for the key, evit one of them and rehash the evicted item. If a cycle found, re-build from scratch with new seed (rare) or rebuild with larger table (common).

## **Dynamic Hashing Schemes**

* Size of hash table is not fixed. Able to resize it according to demand.
* Chained Hashing: each slot is a pointer to a linked list of buckets.
* Extendible Hashing: Multiple slots in the hash table may point to the same bucket chain. More balanced version of chained hashing.
* Linear Hashing: Split buckets with too many elements by adding a new slot entry and creating new hash function.
